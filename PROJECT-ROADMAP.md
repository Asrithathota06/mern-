# Project Roadmap

This workspace contains 14 experiments. The goal is to keep the first 7 polished and easy to run, while the remaining 7 stay clearly separated for your friend.

## Ownership Split

### My side: Experiments 1 to 7

1. Responsive shopping cart web app with registration, login, catalog, and cart pages using CSS3, Flexbox, and Grid.
2. Bootstrap-based responsive version of the shopping web app.
3. Client-side JavaScript validation for the pages from experiments 1 and 2.
4. Weather app using ES6 features, async/await, and OpenWeatherMap data displayed with charts.
5. Java standalone CRUD application connected to MySQL or Oracle.
6. XML bookstore document with DTD and XSD validation.
7. Servlet controller that connects experiment 1 with experiment 5 database.

### Friend's side: Experiments 8 to 14

8. Session tracking mechanisms: Cookies and HTTP Session.
9. Custom Node.js server using `http`, plus `os`, `path`, and `event` modules.
10. Express REST API for student CRUD operations using Postman.
11. JWT-protected endpoints for the Express app.
12. React student management app with registration, login, contact, about, and routing.
13. React weather service using OpenWeatherMap and Chart.js.
14. React TODO app and GitHub deployment.

## First 7 Quality Checklist

- Each experiment should have a clear entry file and a short README or run note.
- Keep naming consistent across folders, pages, and scripts.
- Remove duplicate or unused files if they confuse the main path.
- Make sure every experiment has a working run command documented in its folder.
- Keep experiments 1 to 7 focused on the assignment requirements only.

## Suggested Cleanup Order

1. Lock experiment 1 to 7 folder names and entry points.
2. Add or fix README/run instructions for each first-7 folder.
3. Fix build/run issues in experiment 5 and 7 first, because they depend on Java tooling.
4. Then keep experiments 8 to 14 separate so they do not clutter your side of the workspace.
